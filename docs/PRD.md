# 守岗 SafePost 产品需求文档 PRD

## 1. 产品概述

产品名称：守岗 SafePost

产品定位：物流作业人员安全闭环管理系统。

产品愿景：让每一位进入车厢作业的人，都能被看见、被确认、被守护。

产品目标：

- 上岗前可筛查。
- 作业中可监护。
- 异常时可发现。
- 事故后可追溯。

## 2. MVP 核心命题

MVP 不做 AI，不做 GPS，不做摄像头，不做心率和血氧监测。

MVP 只验证一条关键闭环：

进入车厢 → 开始作业 → 模拟失去行动能力 → 静止/无响应 → 系统自动发现 → 企业微信报警 → 主管 2 分钟内确认异常。

## 3. 用户角色

| 角色 | 场景 | 目标 |
| --- | --- | --- |
| 装卸工 | 进入车厢、月台、仓库作业 | 明确是否允许作业，遇险可报警 |
| 班组长 | 分配装车任务，现场巡查 | 知道谁在作业、谁异常、谁超时 |
| 安全主管 | 安全管理和事故处置 | 第一时间收到报警并形成处置记录 |
| 企业管理员 | 管理组织、人员、设备 | 维护设备、账号、规则和数据 |
| 站点负责人 | 评估站点安全水平 | 查看报表和风险趋势 |

## 4. 系统组成

### 4.1 安全工牌

腰夹式，每人唯一绑定。

功能：

- 工牌唯一 ID。
- 身份识别。
- SOS 主动报警。
- 失联检测。
- 长时间静止检测。
- 长时间无响应检测。
- 作业计时。
- 喝水提醒。
- 状态灯显示。
- 震动提醒。
- 蜂鸣提醒。

状态灯：

| 状态 | 灯光 | 含义 |
| --- | --- | --- |
| ALLOWED | 绿灯 | 允许作业 |
| PENDING | 黄灯 | 待检测 |
| FORBIDDEN | 红灯 | 禁止作业 |
| ALERT | 红灯爆闪 | 紧急报警 |

### 4.2 上岗检测站

员工每日上岗前完成：

- 身份确认。
- 血压检测。
- 酒精检测。

检测结果上传云端，并同步至工牌。云端判定允许上岗或禁止上岗。

### 4.3 安全基站

部署位置：

- 装卸口。
- 月台。
- 仓库区域。

功能：

- 工牌通信。
- 广播通知。
- 作业区域管理。
- 报警接收。
- 数据上传。

### 4.4 云平台

功能：

- 人员档案管理。
- 工牌管理。
- 基站管理。
- 检测记录管理。
- 作业记录管理。
- 报警记录管理。
- 风险记录管理。
- 数据分析。

## 5. 业务流程

### 5.1 上岗准入流程

1. 工人领取或佩戴唯一工牌。
2. 在检测站完成身份确认。
3. 完成血压检测。
4. 完成酒精检测。
5. 检测结果上传云端。
6. 云端根据规则判断是否允许作业。
7. 判定结果写入工牌。
8. 工牌显示绿灯、黄灯或红灯。
9. 后台记录检测结果和准入状态。

### 5.2 作业监护流程

1. 工人进入车厢或作业区域。
2. 基站识别工牌进入区域。
3. 系统创建作业会话。
4. 工牌周期上报心跳、电量、运动状态和响应状态。
5. 系统计算作业时长、静止时长、失联时长和无响应时长。
6. 达到规则阈值时触发提醒或报警。
7. 报警同步到后台和企业微信。
8. 主管确认并处置。

### 5.3 SOS 报警流程

1. 工人长按 SOS 按键。
2. 工牌红灯爆闪、震动或蜂鸣。
3. 工牌通过基站或 MQTT 上报 SOS。
4. 云端创建 CRITICAL 报警。
5. 企业微信通知主管。
6. 后台显示报警并记录处置动作。

## 6. 功能需求

| 编号 | 功能 | 描述 | 优先级 |
| --- | --- | --- | --- |
| F01 | 工牌唯一 ID | 每个工牌有唯一编号并可绑定人员 | P0 |
| F02 | SOS 报警 | 工牌按键主动触发紧急报警 | P0 |
| F03 | 静止检测 | 工牌检测长时间无运动 | P0 |
| F04 | 无响应检测 | 工牌或基站发起确认，超时未响应报警 | P0 |
| F05 | 作业计时 | 记录进入作业区域后的持续时长 | P0 |
| F06 | 喝水提醒 | 按规则下发提醒并本地震动/蜂鸣 | P1 |
| F07 | 基站接收 | 基站接收工牌心跳和报警 | P0 |
| F08 | 后台显示 | 看板展示工牌、作业、报警状态 | P0 |
| F09 | 企业微信报警 | 报警推送到主管企业微信群或应用 | P0 |
| F10 | 检测站接入 | 血压和酒精检测上传云端 | P1 |
| F11 | 准入状态灯 | 绿/黄/红/爆闪表达状态 | P0 |
| F12 | 报警日志 | 记录报警、确认、关闭和备注 | P0 |

## 7. 报警规则

| 规则 | 默认阈值 | 级别 |
| --- | --- | --- |
| SOS 主动报警 | 立即触发 | CRITICAL |
| 工牌失联 | 120 秒无心跳 | CRITICAL |
| 长时间静止 | 120 秒无显著运动 | WARN |
| 无响应 | 180 秒未确认 | CRITICAL |
| 作业超时 | 45 分钟 | WARN |
| 喝水提醒 | 30 分钟 | INFO |
| 低电量 | 20% 以下 | WARN |

## 8. 非功能需求

- 报警延迟：SOS 到后台入库小于 3 秒。
- 通知延迟：报警到企业微信小于 10 秒。
- 处置目标：主管 2 分钟内确认异常。
- 可靠性：MQTT 使用 QoS 1。
- 续航目标：工牌单次充电支持至少 1 个班次。
- 安全性：设备密钥独立签发，后台使用账号权限管理。
- 可追溯：所有检测、状态变更、报警、处置动作留痕。

## 9. 后台页面

| 页面 | 功能 |
| --- | --- |
| 安全看板 | 今日上岗、在线率、待处理报警、异常趋势 |
| 工牌管理 | 工牌绑定、状态、电量、最后在线 |
| 人员管理 | 员工档案、班组、工牌绑定 |
| 检测记录 | 血压、酒精、检测结果、准入状态 |
| 作业记录 | 作业区域、开始时间、持续时长、结束时间 |
| 报警日志 | 报警级别、类型、位置、处置状态 |
| 规则配置 | 失联、静止、无响应、提醒阈值 |

## 10. 验收标准

- 工牌可以唯一识别并显示状态灯。
- 工牌 SOS 可触发后台报警和企业微信通知。
- 模拟静止和无响应可触发自动报警。
- 后台可以展示报警并完成确认。
- 报警日志可追溯。
- 基站能接收工牌消息并向云端上传。
- MVP 试点中，关键异常能在 2 分钟内被主管确认。
