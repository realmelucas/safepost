# 事故起源

## 背景

2023 年夏天，某物流转运中心发生严重生产事故。一名装卸工在高温环境下独自在车厢内装车，现场监管不足，车厢内部又属于传统视频监控盲区。作业人员失去行动能力后未被及时发现，皮带机持续向车厢输送货物，最终造成严重后果。

## 暴露的问题

1. 车厢内作业存在天然监控盲区。
2. 单人进入车厢作业后，现场缺少持续生命状态确认。
3. 上岗前健康与酒精状态检查依赖人工流程，记录不稳定。
4. 现场异常缺少自动升级机制，主管无法第一时间收到报警。
5. 安全记录分散，事故复盘缺少可信时间线。

## 产品动机

SafePost 的目标不是替代现场管理，而是为车厢装卸、转运中心、仓储月台等高风险岗位增加一道低成本、可追溯、能主动报警的安全监护层。

核心原则：

- 每个上岗人员都绑定唯一安全工牌。
- 上岗资格由检测结果、云端策略和现场状态共同决定。
- 人在车厢内作业时必须持续可见、可确认、可报警。
- 所有检测、通知、报警和处置都形成日志。

## 设计边界

SafePost 不直接控制皮带机、装卸设备或门禁设备，但可以通过 API、继电器网关或工业协议适配模块对接现场系统。初版以准入判断、人员状态监护、异常告警和管理后台为核心。
